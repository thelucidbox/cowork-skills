# Knowledge Capture — Output Folder

## What Appears Here

```
output/
├── kb-article.md       # Full article
├── quick-reference.md  # TL;DR version
└── related-articles.md # Cross-references
```

---

## File Descriptions

### kb-article.md
Complete knowledge base article:
- Problem description
- Symptoms and context
- Root cause
- Step-by-step solution
- Gotchas and failed attempts
- Tags and references

### quick-reference.md
Condensed version:
- One-line solution
- Minimal steps
- Link to full article

### related-articles.md
Cross-reference suggestions:
- Similar problems
- Related topics
- Suggested links

---

## Using the Outputs

### Adding to Your KB
1. Review `kb-article.md` for accuracy
2. Copy to your knowledge base location
3. Update cross-references
4. Add to index/search

### Distribution
- Share `quick-reference.md` in Slack
- Link full article for complex issues
- Add to onboarding materials if relevant

---

## Building Institutional Knowledge

Save articles systematically:
```
knowledge-base/
├── category/
│   └── article.md
└── index.md
```

Over time, this becomes your team's institutional memory.
