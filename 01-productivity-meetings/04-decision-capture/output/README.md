# Decision Capture — Output Folder

## What Appears Here

```
output/
├── decision-framework.md   # Complete analysis
├── options-matrix.md       # Side-by-side comparison
├── assumptions.md          # Surfaced assumptions
└── decision-record.md      # Archive after deciding
```

---

## File Descriptions

### decision-framework.md
Complete decision analysis:
- Situation and context
- Stakeholders and constraints
- Weighted criteria
- Options with scores
- Recommendation with rationale

### options-matrix.md
Quick-reference comparison:
- All options side by side
- Key factors compared
- Best option per factor

### assumptions.md
Hidden assumptions surfaced:
- Stated assumptions with validation status
- Discovered assumptions
- Risk if assumptions wrong

### decision-record.md
Archive-ready template:
- Fill in after decision made
- Captures final choice and rationale
- For future reference

---

## Using the Outputs

### During Decision Process
1. Review `decision-framework.md` thoroughly
2. Validate `assumptions.md` — are they accurate?
3. Share `options-matrix.md` for stakeholder input

### After Deciding
1. Update `decision-record.md` with final choice
2. Archive in decisions folder
3. Set calendar reminder for review trigger

### For Future Reference
Save to:
```
decisions/
└── [date]-[decision-name]/
    ├── decision-framework.md
    └── decision-record.md
```

---

## Iteration

If framework doesn't feel right:
- Ask to adjust criteria weights
- Add options not initially considered
- Surface additional assumptions
- Request different recommendation depth
