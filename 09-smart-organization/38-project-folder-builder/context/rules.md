# Project Folder Builder — Rules & Guidelines

## Standard Templates

### Software Project
```
project-name/
├── README.md
├── docs/
│   ├── requirements/
│   └── architecture/
├── src/
├── tests/
├── config/
├── scripts/
├── assets/
└── dist/
```

### Client Project
```
client-project/
├── README.md
├── 01-Brief/
├── 02-Research/
├── 03-Concepts/
├── 04-Development/
├── 05-Review/
├── 06-Delivery/
├── _Feedback/
└── _Admin/
```

### Research Project
```
research-project/
├── README.md
├── literature/
├── data/
│   ├── raw/
│   └── processed/
├── analysis/
├── outputs/
└── references/
```

## Naming Conventions

### Folders
- Use kebab-case or numbers prefix
- Descriptive but concise
- Consistent across project

### Files
- Clear purpose in name
- Include dates where relevant
- Version if needed (v1, v2)
