# Kickoff Kit — Output Folder

## What Appears Here

```
output/
├── project-charter.md        # Project definition
├── stakeholder-map.md        # Who's involved
├── risk-register.md          # Risks and mitigations
└── milestone-1-breakdown.md  # First phase tasks
```

---

## File Descriptions

### project-charter.md
Complete project definition:
- Goals and success metrics
- Scope (IN and OUT)
- Constraints
- Milestones

### stakeholder-map.md
Everyone involved:
- Stakeholder profiles
- RACI matrix
- Communication plan

### risk-register.md
Risk management:
- Identified risks
- Probability and impact
- Mitigations and contingencies

### milestone-1-breakdown.md
First phase in detail:
- Task breakdown
- Owners and estimates
- Dependencies
- Definition of done

---

## Using the Outputs

### Kickoff Meeting
1. Review charter with stakeholders
2. Validate stakeholder map
3. Discuss risks openly
4. Align on Milestone 1 plan

### Ongoing Project
- Update risk register as new risks emerge
- Reference charter for scope questions
- Use milestone breakdown for sprint planning

---

## Post-Kickoff

After approval:
- Store charter as project source of truth
- Set up risk review cadence
- Create Milestone 2 breakdown when approaching
